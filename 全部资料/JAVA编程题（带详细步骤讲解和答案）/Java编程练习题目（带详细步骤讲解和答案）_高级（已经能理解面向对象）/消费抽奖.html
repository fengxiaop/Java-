1、实验要求

    本实验要求：以现实生活中消费抽奖为背景，体验集合和映射的实际应用特性。

    1-1. 业务说明：

        1-1.1. 本实验以实际生活中消费抽奖为业务背景。

        1-1.2. 为保证奖品的有限性和等级性，只提供4类奖品。 

        1-1.3. 每类奖品的数量为有限个，统一放到奖池中供抽取。

        1-1.4. 每类奖品对应唯一的键值索引（抽奖号），根据索引值取得奖品项。

        1-1.5. 每类奖品的数量被抽取一件，其总数就减少一件。

        1-1.6. 本业务关键概念：奖项、奖池以及抽奖。

2.代码清单：cn.campsg.java.experiment.entity.DrawReward.java

public class DrawReward {

 /**

  * Integer为抽奖号，Awards为抽到的奖项

  */

  private Map<Integer, Awards> rwdPool = null;

  public DrawReward() {

    this.rwdPool = new HashMap<Integer, Awards>();

    // 初始化奖池：创建1个特等奖、3个一等奖、6个二等奖、其它奖100

    rwdPool.put(8, new Awards("特等奖：￥5000", 1));

    rwdPool.put(1, new Awards("一等奖：华为Mate X", 4));

    rwdPool.put(2, new Awards("二等奖：格力吸尘器", 6));

    rwdPool.put(0, new Awards("阳光普照奖：家庭大礼包", 100));

  }

 

  /**

   * 模拟抽奖:draw award

   * @param rdKey 抽奖号，即中奖号码

   */

  public void draward(int rdKey) {

   Awards aw = this.rwdPool.get(rdKey);  

  if(aw==null)

        return ;

    if (aw.getCount() < 1) {

       System.out.println(aw.getName() + "，该奖项已抽完。");

       return;

    }else{

      // 更新奖项剩余数量

      aw.setCount(aw.getCount() - 1);

      System.out.println("抽奖结果：" + aw.getName());

    }

 }

  /**

   * 展现抽奖后奖池剩余奖项情况

   */

  public void showSurplus() {

    Iterator<Awards> itor = rwdPool.values().iterator();

    while (itor.hasNext()) {

    Awards aw = itor.next();

      System.out.println(aw.getName() + ";剩余奖项数量：" + aw.getCount());

    }

  }

}
